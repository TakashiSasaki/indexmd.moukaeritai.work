# Agent Workflows & Procedures

This document outlines the standard operating procedures for all AI coding agents (Jules, Codex, Copilot, etc.) contributing to this repository.

## 1. Branch Strategy
- `main`: The absolute source of truth. Usually updated directly via Google AI Studio.
- `jules/integration`: The auditing and integration branch for Google Jules.
- **Ephemeral Branches**: Codex, Copilot, or human developers should create short-lived feature branches originating from `main` or `jules/integration`.

## 2. Merge Conflict Resolution
Because multiple agents edit code, conflicts between `main` and `jules/integration` are inevitable.
- **Source of Truth**: In case of a logic conflict, `main` (the AI Studio output) takes precedence.
- **Protection**: Do **NOT** force-push or reset `jules/integration` to match `main`.
- **Resolution**: Use standard `git merge`. If conflicts arise, manually resolve them. Pay special attention to:
  - `package.json` & `package-lock.json` (do not blindly overwrite lockfiles).
  - Schema versions (`schemas/*.json`).
  - `AGENTS.md` (do not delete security constraints during conflict resolution).

## 3. Pull Request Guidelines (for Copilot / Codex)
- **Scope**: Keep PRs small and focused. Separate documentation updates from logic changes.
- **Description**: Autogenerated PR descriptions must include a section explicitly stating:
  - What was verified locally (e.g., `npm run test:unit` passed).
  - What could not be verified and why (e.g., "Could not verify OAuth flow because UI preview is unavailable").

## 4. Verification Requirements Before Commit
Every agent must execute the following before committing to the repository, unless the environment strictly prevents it:
1. `npm run lint`
2. `npm run test:unit`
3. `npm run build`

If a step cannot be executed, the agent must document the reason in the commit message or PR summary.

## 5. Documentation Updates vs. Code Updates
- If an agent discovers a mismatch between documentation (like `AGENTS.md`) and actual code behavior, the **code** is usually correct. Update the documentation to reflect the codebase, unless the code violates a Hard Safety Constraint listed in `AGENTS.md`.
