# AI Coding Agent Prompt: Document & Schema Consistency Check

以下のプロンプトは、コーディングエージェント（AIアシスタント）に対して、定期的なドキュメントとコードの整合性チェック・更新を依頼するためのテンプレートです。新しい機能追加やスキーマの変更があった後などに、このプロンプトをエージェントに送信して実行させてください。

---

## 📋 プロンプト・テンプレート

```text
【目的】
本プロジェクトのドキュメント（スキーマ定義、設計方針、READMEなど）と、実際のコード実装（型定義、バリデーションロジック、テストコード）の間に齟齬がないか確認し、整合性を保つための保守作業をお願いします。

【確認スコープ】
1. スキーマ定義・型定義
  - `src/lib/summaryAnalysisSchema.ts` などの主要なスキーマ定義ファイルと、関連する型定義(`src/types.ts` 等)を比較し、不足しているフィールドや不正な型がないか確認してください。
  - スキーマのバージョン（例: `SUMMARY_ANALYSIS_SCHEMA_VERSION`）が最新の意図に沿っているか確認してください。

2. バリデーションとテスト
  - `src/lib/summaryAnalysisSchema.test.ts` などのテストファイルが、現在のスキーマ実装を網羅的にテストできているか確認してください。
  - 新しく追加されたスキーマのプロパティや制約（例: Enumの追加、必須フィールドの変更）に対するテスト漏れがあれば、テストコードを補充してください。

3. プロンプト生成ロジック
  - `src/lib/promptSpecs.ts` などのプロンプト生成関数が、最新のスキーマ定義と矛盾する指示を出していないか確認してください。（例: 旧フィールド名を使用していないか等）

4. 関連ドキュメントの更新
  - プロジェクト内の関連ドキュメント（`docs/`ディレクトリ等にあるスキーマ変更履歴や実験ガイド）を読み込み、実際のコードの挙動と異なっている箇所があれば修正を提案・実行してください。
  - `AGENTS.md` に記載されているルール（Firestoreの書き込み制限やGeminiフォールバックエンジンの仕様など）が守られているか、該当する最新コードのチェックを行ってください。

【実行手順】
1. 上記の各ファイルを `view_file` または `shell_exec` ツールを用いて読み込み、現状の差分や不整合を分析してください。
2. 修正すべき点（不整合、テスト不足、ドキュメントの陳腐化など）をリストアップしてください。
3. リストアップした修正点について、`edit_file` を用いてコードまたはドキュメントを更新してください。
4. 修正完了後、必ず `npm run lint` と `npm run test:unit` を実行して、エラーが発生しないことを確認してください。
5. 最後に、どのような不整合を発見し、どのように修正したかのサマリーを報告してください。
```

---

## 💡 使い方
- **タイミング**: 大きなプルリクエストのマージ後、または新しいスキーマバージョン（例: `v1.1.0-draft.1` から `v1.1.0` への移行時）の策定後に実行するのが最適です。
- **カスタマイズ**: プロンプト内の `【確認スコープ】` のファイルパスは、今後のプロジェクト構成の変更に合わせて適宜修正してください。
